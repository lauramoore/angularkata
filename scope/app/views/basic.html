Basic Scope

"Scope is not the model.. it is just a reference to what's happening [inside the dom]". So scope is what you assign models to, not the model itself. egghead.io tutorial

Root Scope

Root scope sits at the top of the scope inheritance, changes to this model can be reflected across all other application scopes which are children of the root scope.

Plain Variable
{{plainvar}}
Model Variable
{{model.var}}

Controller Scope

Controller scopes are children of root scope, and variables not found local will be pulled from parent scopes all the way back to root.

Plain Variable
{{plainvar}}

Plain variables are limited to current scope, inheritance stops as soon as variable is declared in the controller scope

Model Variable
{{model.var}}

Model variables maintain inheritance, as it's the model value that changes not scope reference, and the model is declared at root scope.
